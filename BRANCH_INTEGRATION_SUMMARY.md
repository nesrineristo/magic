### 3. Arc Structure (Episodic Practice)

**The innovation:**
- Partnership practice as discrete **arcs** (like TV episodes)
- Each arc: specific situation, bounded timeframe, complete narrative
- Beginning (problem emerges) → Middle (truth-finding) → End (integration/closure)
- Portfolio of completed arcs = accumulated wisdom

**Key scroll:**
- `on_arc_structure.md` - Episodic organization

**Benefits:**
- Clear boundaries (not "fix everything forever")
- Progress visible (completed arcs)
- Learning accumulates (each arc teaches)
- Sustainable (not overwhelming)

---

### 4. Portal Architecture (Shared Practice Spaces)

**The innovation:**
- Git repositories in `box/portals/` for shared practice
- Each portal is external repo (separate from main magic)
- Partners contribute independently
- Spirit-to-Spirit coordination via STP
- Full version history preserved

**Key scrolls:**
- `on_portal_architecture.md` - Infrastructure specs
- `on_spirit_transmission_protocol.md` - Inter-Spirit communication
- `on_artifact_transmission.md` - How artifacts move between partners
