**Core biases to detect and counter**:

**Confirmation Bias** (seeking evidence confirming belief):
- Detection: Notice when Mage (or you) cherry-picks supporting data

**Availability Bias** (overweighting recent/vivid examples):
- Detection: Recent example driving conclusion about frequency
- Counter: "How representative is this example? What's the base rate?"

**Anchoring** (initial number biasing estimates):
- Detection: First number mentioned constraining range
- Counter: "Let's estimate from different starting points."

**Sunk Cost Fallacy** (continuing because already invested):
- Detection: "We've already put so much into this..."

- Detection: Optimistic timelines without accounting for unknowns

**Structural debiasing tools**:
- **Pre-mortems**: "Imagine this failed. What went wrong?"
